# 회전 버그 수정 — 실무 요약 (먼저 읽기)

> 이 문서는 **결론만 빠르게**. 상세는 `ROTATION_FIX_DESIGN.md`(수정 설계) / `ROTATION_FIX_RISKS.md`(리스크).
> 두 증상: ① 사각형 map 모드에서 회전이 반대로, ② 일정 이상 돌리면 반대로 튐.

---

## 30초 요약

- **원인**: "거울"(화면 Y 뒤집힘)을 좌표에서 한 번 처리하지 않고, 각도에 `-` 붙여 도형마다 땜질했다. + 회전을 "얼마나 움직였나 더하기"로 계산해 각도 점프(359°→0°)에서 튄다.
- **고치는 법**: ① 거울은 **한 곳에서만** 처리(각도 `-` 땜질 제거), ② 회전은 **지금 가리키는 절대 각도**를 그대로 쓰기.
- **함정**: 사각형만 고치면 나머지가 거꾸로 깨진다. **다 같이** 고쳐야 한다.

---

## 위험도 순위 + 한 줄 대응 (이 순서로)

### 0순위 💾 저장/복원 호환성 — 배포 리스크 (제일 먼저 챙겨라)
- **왜**: serialize는 **visual rotation 값**을 저장, deserialize는 그걸 `setAngle`로 복원. 정준 각도를 바꾸면 **기존 저장 파일이 반대로 열린다.** 신규 도형 테스트로는 절대 못 잡고, 배포 후 되돌리기 어렵다.
- **대응**: 직렬화에 **버전 필드** + 로드 시 **구버전 1회 마이그레이션**(옛 visual-angle → 새 정준-angle). **수정 전 저장 파일 라운드트립**으로 "수정 전과 같은 방향으로 열리는지" 확인.

### 1순위 ⚠️ arc(호) — 시각 정확성에서 가장 위험
- **왜**: 깨져도 **에러 없이 그럴듯하게** 그려져 리뷰에서 놓침. map setAngle 경로가 불규칙하고 내부 각도(start/sweep/볼록)가 많음.
- **대응**: 검증 1순위를 **사각형이 아니라 arc로**. start 점 위치 + 벌어지는 방향을 골든 스크린샷으로 박고, **arc 통과 전 배포 금지**.
- **주의**: **sector(부채꼴)는 arc와 동일 위험 아님.** sector는 map setAngle이 더 정규적인 delta라 **상대적으로 덜 위험**. 단 angle-box 계열이라 **검증은 arc와 같이** 한다. (수정 우선순위: arc > sector)

### 2순위 🖱️ 맵 틸트 가이드/bbox — 가장 깨지기 쉬운 UI
- **왜**: 빌드는 멀쩡, **사용자가 지도 기울이고 줌하는 순간에만** 터짐 → 자동 테스트로 잡기 어려움.
- **대응**: 가이드 그리는 코드는 그대로 두고 **새 각도 값만 흘려넣기**. 틸트 상태를 **초반부터 자주** 확인. `map.project` 기반 유지(원근 처리됨), **손으로 만든 평면 행렬로 바꾸지 말 것**.

### 3순위 🧭 회전 경로가 문서보다 많음 — 범위 추적이 진짜 골치
- **왜**: grep으로 다 찾은 줄 알아도 **숨은 경로**가 남음. 특히 **"단일 선택" 안에도** rectangle-screen / polygon-bounds / ellipse-map-tilted **하위 분기**가 더 있어서, 그룹만 고치면 **단일은 여전히 반대로 돈다.**
- **대응(핵심 트릭)**: 각도가 **최종 적용되는 단 한 곳**(렌더 직전)에 체크포인트 + "누가 세팅했는지" 로그를 박는다. → 모르던 경로가 회전될 때마다 **런타임이 스스로 정체를 드러냄**. fast path 최적화도 이 한 곳(`applyRotation` 등)을 **반드시 거치게**(빠르게 = 변환 건너뛰기 금지).

## 한 줄 결론

> **각도에 `-` 땜질하던 걸 다 걷어내고 "거울 방향 + 회전 방식"을 한 군데서 통일한다.**
> **단, 사각형만 고치지 말고 모든 도형·모든 회전 경로(단일 내부 분기 포함)를 같이 고치되, 선택상자(가이드) 그리는 방식은 그대로 두고, 기존 저장 데이터는 버전 마이그레이션으로 흡수한다.**
> 순서: **⓪ 저장 마이그레이션 먼저 → ① arc부터 검증(sector는 같이) → ② 틸트 UI 초반부터 계속 확인 → ③ 회전 경로를 한 곳으로 강제해 숨은 단일 분기를 런타임이 잡게.**

## 쉬운 비유 (이해용)

- **증상 ①**: 거울 앞에서 오른손 들면 거울 속은 왼손. 지도 화면이 그 거울이라 "시계방향"이 화면에선 반시계로 보임 → 개발자가 각도에 `-`를 붙여 땜질.
- **증상 ②**: 시계 초침이 59초→0초로 확 뛰듯, 각도도 359°→0°로 점프. "얼마나 움직였나 더하기" 방식이라 이 점프에서 홱 돌아감.
- **리스크 1**: 사각형은 "몇 도 돌았나" 하나뿐, 부채꼴은 "어디서 시작·어느 쪽으로 얼마나" 각도가 여럿 → 사각형 땜질만 떼면 부채꼴이 거꾸로.
- **리스크 2**: 지도를 비스듬히 보면 화면이 사다리꼴로 찌그러지고 선택상자는 그걸 감안해 그려짐. 억지로 한 계산법으로 합치면 그 보정이 사라져 선택상자가 떨어져 나감.
- **리스크 3**: 회전 코드가 "그룹/단일/옛날" 따로라, 그룹만 고치면 단일은 여전히 버그 → "어떨 땐 되고 어떨 땐 안 됨".
