# 그룹 도형 프로젝트 AI에게 던질 질문지 (현재 구현 설계 확인용)

> **용도**: 이 블록을 그대로 그쪽 프로젝트 AI에게 붙여넣어 **지금 어떻게 구현돼 있는지** 답을 받는다.
> 받은 답을 `SELECTION_BOX_CHECKLIST.md`(우리 레퍼런스 규약)와 대조해 우리가 수정한다.
> 그쪽 AI에게는 **장황한 설명 말고 코드 한 줄/공식/예/아니오** 위주로 간결히 답해달라고 요청한다.

## 붙여넣기용 프롬프트 (여기부터 복사)

아래 질문에 **각 1~3줄, 가능하면 실제 코드/공식**으로 간결히 답해줘. 추측이면 "추측"이라고 표시해줘.

**A. 환경**
1. `maplibre-gl`(또는 사용 중인 지도 라이브러리) 버전은? custom layer `render` 시그니처는 `render(gl, matrix)`인가 `render(gl, args)`인가?
2. 도형/그룹의 내부 좌표계 Y축은 **위로 +** 인가 **아래로 +** 인가? (Mercator Y-down 그대로 쓰는지)

**B. 데이터 모델**
3. 그룹 회전 상태는 어디에 저장돼 있나? (`rotation` 한 값 vs 자식마다 각자) 자식 도형 정점은 **그룹 로컬 좌표**로 저장하나, **절대 지도좌표**로 저장하나?
4. 그룹 회전 피벗(중심)은 무엇인가? 그룹 바운딩 중심인가, 자식 개별 중심인가?

**C. 회전 측정 (핸들 드래그 → 각도)**
5. 회전 각도를 `atan2`로 잴 때, **어느 좌표에서** 재나? (screen / local / mercator) — 그 한 줄 코드 붙여줘.
6. 측정한 회전값을 **매 프레임 절대 각도로 set** 하나, 아니면 **delta를 누적**(`rotation += Δ`)하나? — 해당 코드 붙여줘.
7. 각도 정규화 함수가 있나? 있으면 식이 `((v+180)%360+360)%360-180` 형태(이중 모듈로)인가, 단순 `v%360`인가?

**D. 핵심 의심 지점 — angle 반전**
8. **map 모드에서 angle을 반전**하는 코드(`mapMode ? -a : a`, `angle *= -1`, `-rotation`, `flipY ? -θ : θ` 등)가 있나? 있으면 **정확한 위치와 조건**을 붙여줘.
9. 그 반전이 **map 모드에만** 걸려 있나, 스크린 모드에도 걸려 있나?

**E. Y축 반사 횟수 (가장 중요)**
10. **측정 → 저장 → 적용 → 렌더** 파이프라인 전체에서 Y 부호를 뒤집는 곳(`-y`, `scale(s, -s, s)`, `scaleY(-1)`, CSS `transform: scaleY(-1)` 등)이 **각각 어디에 몇 번** 있나? 전부 나열해줘.
11. 회전 각도를 **재는 프레임**과 **렌더하는 프레임**의 Y 반사 횟수가 같나? (같으면 부호 정상, 다르면 어긋남)

**F. 이동(translate)**
12. 도형/그룹 이동을 **Mercator delta**로 하나, **local delta**로 하나? 드래그 시작 시 시작 포인터와 시작 중심을 저장하나?
13. 이동/회전에서 경도 ±180°(antimeridian)나 `atan2` ±180° 경계 가드가 있나?

**G. 오버레이 ↔ 실제 도형**
14. 선택상자(핸들) 좌표와 실제 렌더되는 도형이 **같은 변환 함수**를 쓰나, 따로 계산하나?
15. 드래그/렌더 핸들러가 최신 상태를 **ref(즉시값)**로 읽나, **state(클로저)**로 읽나?

(여기까지 복사)

## 우리가 답을 받으면 보는 포인트 (내부 메모)

받은 답을 이렇게 해석해 수정 지점을 정한다. (상세 근거는 `SELECTION_BOX_CHECKLIST.md`)

- **8·9·10·11 → angle 반전의 진짜 원인**: 반사가 홀수면 `-angle`이 "필요해 보이는" 구조. 답에서 Y반사 개수가 측정/렌더 간 다르면 → 우리는 **각도 보정(`-angle`)을 떼고 좌표 측정 단계에서 Y를 한 번만 뒤집는** 방향으로 수정. (체크리스트 8-A 증상①)
- **6·7 → 오른쪽→왼쪽 튐**: delta 누적 + 단순 `%360`이면 ±180 경계 점프가 원인. → **절대 각도 set + 이중 모듈로 정규화**로 수정. (8-A 증상②)
- **3·4 → 그룹 흩어짐/피벗 오류**: 자식이 절대좌표 저장이거나 피벗이 자식 중심이면 → **그룹 로컬 좌표 + 그룹 중심 피벗**으로. (1·2·8장)
- **12 → 이동 드리프트**: local delta면 회전 상태에서 틀어짐 → **Mercator delta + 시작값 저장**으로. (7장)
- **1 → 안 보임/통째 어긋남**: maplibre 버전이 v5거나 시그니처가 다르면 매트릭스 처리부터 수정. (9장)
- **14·15 → 선택상자-도형 불일치/한 프레임 지연**: 변환 분리 또는 state 클로저면 → **동일 변환 공유 + ref 읽기**. (5·10장)
